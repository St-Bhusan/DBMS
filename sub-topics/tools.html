DBMS Tools

1. Server: XAMPP Server

  • XAMPP is a free and open-source cross-platform web server solution stack package.
  • It includes Apache HTTP Server, MariaDB (or MySQL), and interpreters for scripts written in PHP and Perl.
  • Used for local development and testing of web applications that require a database.
  • Provides an easy way to set up a local server environment for database management and web development.

2. SQL (Structured Query Language)

  • SQL is a standard language for managing and manipulating relational databases.
  • It is used to perform tasks such as querying data, updating records, and managing database structures.

2.1 DDL (Data Definition Language)

  • Used to define and modify the structure of database objects (tables, schemas, etc.).
  • Common DDL commands:
    • CREATE: Creates a new table or database.
    • ALTER: Modifies an existing database object.
    • DROP: Deletes tables or databases.
    • TRUNCATE: Removes all records from a table, but not the table itself.

2.2 DML (Data Manipulation Language)

  • Used to manipulate data stored in database tables.
  • Common DML commands:
    • INSERT: Adds new records to a table.
    • UPDATE: Modifies existing records.
    • DELETE: Removes records from a table.

2.3 DQL (Data Query Language)

  • Used to query and retrieve data from databases.
  • Main DQL command:
    • SELECT: Retrieves data from one or more tables.

2.4 DCL (Data Control Language)

  • Used to control access to data in the database.
  • Common DCL commands:
    • GRANT: Gives user access privileges to the database.
    • REVOKE: Removes user access privileges.

2.5 SQL Joins

  • Used to combine rows from two or more tables based on a related column.
  • Types of joins:
    • INNER JOIN: Returns records with matching values in both tables.
    • LEFT JOIN (LEFT OUTER JOIN): Returns all records from the left table, and matched records from the right table.
    • RIGHT JOIN (RIGHT OUTER JOIN): Returns all records from the right table, and matched records from the left table.
    • FULL JOIN (FULL OUTER JOIN): Returns all records when there is a match in either left or right table.
  • Joins are essential for querying data spread across multiple tables.
