SQL Constraints: Definition, Types, and In-depth Explanation

What are SQL Constraints?

SQL constraints are rules enforced on columns in a table to ensure the accuracy, reliability, and integrity of the data stored in a relational database. Constraints restrict the type of data that can be inserted, updated, or deleted from a table, thereby maintaining the consistency of the database.

Types of SQL Constraints

  • NOT NULL
  • UNIQUE
  • PRIMARY KEY
  • FOREIGN KEY
  • CHECK
  • DEFAULT

Detailed Explanation of Each Constraint

1. NOT NULL Constraint

The NOT NULL constraint ensures that a column cannot have a NULL value. This means every row in the table must have a value for this column. It is commonly used for columns that require mandatory data, such as user names or IDs. 

CREATE TABLE Employees (
    EmployeeID INT NOT NULL,
    Name VARCHAR(100) NOT NULL
);

2. UNIQUE Constraint

The UNIQUE constraint ensures that all values in a column or a set of columns are distinct from each other. Unlike the PRIMARY KEY, a table can have multiple UNIQUE constraints, and columns with UNIQUE constraints can accept NULL values (except in some databases). 

CREATE TABLE Employees (
    Email VARCHAR(255) UNIQUE
);

3. PRIMARY KEY Constraint

The PRIMARY KEY constraint uniquely identifies each record in a table. It combines the properties of NOT NULL and UNIQUE, meaning no two rows can have the same primary key value, and it cannot be NULL. Each table can have only one primary key, which may consist of single or multiple columns (composite key). 

CREATE TABLE Employees (
    EmployeeID INT PRIMARY KEY
);

4. FOREIGN KEY Constraint

The FOREIGN KEY constraint establishes a relationship between two tables by linking a column (or columns) in one table to the PRIMARY KEY or UNIQUE key in another table. It ensures referential integrity, meaning the value in the foreign key column must exist in the referenced table. 

CREATE TABLE Departments (
    DepartmentID INT PRIMARY KEY
);

CREATE TABLE Employees (
    EmployeeID INT PRIMARY KEY,
    DepartmentID INT,
    FOREIGN KEY (DepartmentID) REFERENCES Departments(DepartmentID)
);

5. CHECK Constraint

The CHECK constraint enforces a condition that each row must satisfy. It can be used to limit the range of values that can be placed in a column, such as ensuring an age is greater than zero or a salary is within a certain range. 

CREATE TABLE Employees (
    Age INT CHECK (Age >= 18),
    Salary DECIMAL(10,2) CHECK (Salary > 0)
);

6. DEFAULT Constraint

The DEFAULT constraint assigns a default value to a column if no value is specified during an insert operation. This helps in ensuring that a column always has a value, even if the user does not provide one. 

CREATE TABLE Employees (
    JoinDate DATE DEFAULT CURRENT_DATE
);

Conclusion

SQL constraints are essential for maintaining data integrity and enforcing business rules at the database level. By carefully applying these constraints, database designers can prevent invalid data entry and ensure the reliability of the database system.
