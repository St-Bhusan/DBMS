Normalization in DBMS

Normalization is a systematic approach of organizing data in a relational database to minimize redundancy and improve data integrity. The main objective of normalization is to divide large tables into smaller, related tables and link them using relationships. This process helps in eliminating duplicate data, ensuring logical data storage, and making the database more efficient.

Why is Normalization Needed?

  • To eliminate redundant (repetitive) data.
  • To ensure data dependencies make sense (only storing related data in a table).
  • To protect the database against anomalies during insert, update, or delete operations.
  • To improve data consistency and integrity.

Types of Normal Forms

Normalization is carried out in several steps, each step corresponding to a "normal form". The most commonly used normal forms are:

1. First Normal Form (1NF)

  • Ensures that each column contains only atomic (indivisible) values.
  • Each record needs to be unique.
  • No repeating groups or arrays are allowed in a table.

Example: A table with multiple phone numbers in a single field violates 1NF.

2. Second Normal Form (2NF)

  • Table must be in 1NF.
  • All non-key attributes must be fully functionally dependent on the entire primary key (no partial dependency).
  • Applies mainly to tables with composite primary keys.

Example: If a table's non-key attribute depends only on part of a composite key, it violates 2NF.

3. Third Normal Form (3NF)

  • Table must be in 2NF.
  • There should be no transitive dependency for non-prime attributes (non-key attributes should not depend on other non-key attributes).

Example: If a table has columns A (primary key), B, and C, and C depends on B, not directly on A, it violates 3NF.

  • Eliminates certain types of anomalies not handled by 3NF.

5. Fourth Normal Form (4NF)

  • Table must be in BCNF.
  • There should be no multi-valued dependencies except a candidate key.
  • Deals with cases where a table contains two or more independent multi-valued facts about an entity.

6. Fifth Normal Form (5NF)

  • Table must be in 4NF.
  • Deals with cases where information can be reconstructed from smaller pieces of information that are stored in the database.
  • Eliminates join dependency anomalies.

Conclusion

Normalization is essential for designing efficient, reliable, and scalable relational databases. By applying various normal forms, database designers can ensure data consistency, reduce redundancy, and prevent anomalies, leading to better data management and performance.
